Follow these steps to properly set up and test your Word add-in.

Step 1: Start Your Servers

Make sure both servers are running:

You should see:

  • ‚úÖ Web server running on port 3000

Step 2: Create a Test Document

Instead of using the old document file:

  1. Open http://localhost:3000/clean-sample-document.html
  2. Copy ALL the content from the white box
  3. Open Microsoft Word
  4. Paste the content
  5. Save as "test-document.docx"

Step 3: Install the Add-in

With your test document open in Word:

  1. Click on the Insert tab
  2. Click Get Add-ins
  3. Click My Add-ins
  4. Click Upload My Add-in
  5. Browse to your project folder: C:\Users\msokrin\Document project
  6. Select manifest.xml
  7. Click Upload

‚úÖ Success Indicators

You should see:

  • A new "Hello Group" section in the Home ribbon
  • A "Hello Moti" button in that section
  • When clicked, a task pane opens on the right side

Step 4: Test the Add-in

In the Word add-in task pane:

  1. Click the Hello Moti button in the ribbon
  2. The task pane should open on the right
  3. Try clicking üîÑ Pull from Word to read the document
  4. Check the Navigation tab for sections

Step 5: Test Web Viewer Sync

Open the web viewer and test sync:

  1. Open http://localhost:3000/web-viewer.html
  2. In Word add-in: Click üîÑ Pull from Word
  3. In web viewer: Click üîÑ Refresh
  4. You should see the document sections appear

‚ö†Ô∏è Troubleshooting

If the add-in doesn't appear:

  • Make sure both servers are running
  • Check if you can access http://localhost:3000/taskpane.html in your browser
  • Try restarting Word and checking the ribbon again

If content is unreadable:

  • Use the clean document from Step 2
  • Make sure you're pasting plain text, not formatted HTML
  • Try creating a completely new Word document

If sync doesn't work:

  • Check the browser console for errors
  • Make sure the API server shows "Updated document data" messages
  • Try refreshing both the web viewer and Word task pane

Step 6: Quick Test Links

Test these URLs to make sure everything is working:

  • Task Pane - Should show the Word add-in interface
  • Web Viewer - Should show the document viewer
  • API Health - Should return {"status":"ok"}
  • Document Sections - Should return the document data
